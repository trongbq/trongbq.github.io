Go database package

23/9/2020

One of the nice thing about Go which I like it very much is that Go has lots of useful built-in packages that help us build software without relying heavily on external packages. Talking about building software, database operations consumes large of our time during development. There are several popular packages can help us reducing development time when working with database like gorm or Facebook ent, but still, working with only built-in database package is enough for most of the cases.

Go has built-in package database/sql which provide a generic interface around SQL database.

Preparation

Bellow is the DDL and model struct that I will use in examples through this post.

DDL (MySQL)

CREATE TABLE `users` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `name` varchar(45) NOT NULL,
  `email` varchar(45) DEFAULT NULL,
  `created_at` timestamp NOT NULL,
  PRIMARY KEY (`id`),
  UNIQUE KEY `id_UNIQUE` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=armscii8

Struct type

type user struct {
	id        int
	name      string
	email     string
	createdAt time.Time
}

func (u user) String() string {
	return fmt.Sprintf("User(id: %v, name: %v, email: %v, created_at: %v)", u.id, u.name, u.email, u.created_at)
}

Establish a database connection

To start, let’s create a sql.DB instance which represents a pool of connections. database/sql package does not include any drivers so in order to use a specific driver, we have to import it ourself. In the example bellow, I use mysql driver from go-sql-driver, when this module is import, it registers to itself to database/sql’s driver list.

// https://golang.org/src/database/sql/sql.go

// Register makes a database driver available by the provided name.
// If Register is called twice with the same name or if driver is nil,
// it panics.
func Register(name string, driver driver.Driver) {
	driversMu.Lock()
	defer driversMu.Unlock()
	if driver == nil {
		panic("sql: Register driver is nil")
	}
	if _, dup := drivers[name]; dup {
		panic("sql: Register called twice for driver " + name)
	}
	drivers[name] = driver
}

// https://github.com/go-sql-driver/mysql/blob/master/driver.go
func init() {
	sql.Register("mysql", &MySQLDriver{})
}

The list of the database drivers can be found here.

package main

import (
	"database/sql"
	"log"

	_ "github.com/go-sql-driver/mysql"
)

func main() {
	db, err := sql.Open("mysql", "root:@tcp(127.0.0.1:3307)/experiment")
	if err != nil {
		log.Fatal(err)
	}
	defer db.Close()

	err = db.Ping()
	if err != nil {
		log.Fatal(err)
	}

	log.Print("Connected")
}

Calling Open may not actually create a connection, we can use Ping to establish a connection to verify if the database connection is alive. For most of the exported methods in database/sql package, there are two versions: one without context and the other has context in its parameters. If we use the first one, context.Background() will be used under the hood.

Working with database

Everytime we call Query or Exec method on *DB, it asks connection poll for a connection which is active but idle or ask the Driver to create a new connection if there are no connections available in the pool.

Query data

Data from database can be queried by using combination of db.Query and rows.Scan function.

db, err := sql.Open("mysql", "root:@tcp(127.0.0.1:3306)/experiment?parseTime=true")
if err != nil {
    log.Fatal(err)
}

log.Print("Get all users data:")
rows, err := db.Query("select * from users")
if err != nil {
    log.Fatal(err)
}
defer rows.Close()

for rows.Next() {
    var u User
    err := rows.Scan(&u.id, &u.name, &u.email, &u.createdAt)
    if err != nil {
	log.Print(err)
    }
    log.Println(u)
}

Execute SQL query

Sometimes, we need to execute some SQLs that do not return rows value like INSERT or DELETE. In this case, we better use Exec method, which returns a Result struct.

// https://golang.org/src/database/sql/sql.go
// A Result summarizes an executed SQL command.
type Result interface {
	// LastInsertId returns the integer generated by the database
	// in response to a command. Typically this will be from an
	// "auto increment" column when inserting a new row. Not all
	// databases support this feature, and the syntax of such
	// statements varies.
	LastInsertId() (int64, error)

	// RowsAffected returns the number of rows affected by an
	// update, insert, or delete. Not every database or database
	// driver may support this.
	RowsAffected() (int64, error)
}

Usage example

db, err := sql.Open("mysql", "root:@tcp(127.0.0.1:3306)/experiment?parseTime=true")
if err != nil {
    log.Fatal(err)
}
r, err := db.Exec("insert into users(name, email) values(?, ?)", "John", "john@gmail.com")
if err != nil {
    log.Fatal(err)
}
insertId, err := r.LastInsertId()
if err != nil {
    log.Fatal(err)
}
rowsAffected, err := r.RowsAffected()
if err != nil {
    log.Fatal(err)
}
log.Printf("Inserted row id: %v", insertId)
log.Printf("Rows affected: %v", rowsAffected)

Statement

Beside using Query and Exec directly, we can also using a prepared statement for later queries or executions. A single prepared statement can be run concurrently.

db, err := sql.Open("mysql", "root:@tcp(127.0.0.1:3306)/experiment?parseTime=true")
if err != nil {
    log.Fatal(err)
}
stmt, err := db.Prepare("insert into users(name, email) values(?, ?)")
if err != nil {
    log.Fatal(err)
}
r, err := stmt.Exec("John", "john@gmail.com")

insertId, err := r.LastInsertId()
if err != nil {
    log.Fatal(err)
}
rowsAffected, err := r.RowsAffected()
if err != nil {
    log.Fatal(err)
}
log.Printf("Inserted row id: %v", insertId)
log.Printf("Rows affected: %v", rowsAffected)

Transactions

Go package database/sql has struct Tx which represent a sql transaction, we can start a transaction by calling BeginTx method or just Begin if you decide to use context.Background() and passing no options. There are only two options for starting a transaction: Isolation level and ReadOnly.

// https://golang.org/src/database/sql/sql.go

// TxOptions holds the transaction options to be used in DB.BeginTx.
type TxOptions struct {
	// Isolation is the transaction isolation level.
	// If zero, the driver or database's default level is used.
	Isolation IsolationLevel
	ReadOnly  bool
}

To ensure data integrity, I recommend to use LevelReadCommited for isolation level for most of the case and seeking for appropriate isolation level when there are some special requirements.

db, err := sql.Open("mysql", "root:@tcp(127.0.0.1:3306)/experiment?parseTime=true")
if err != nil {
    log.Fatal(err)
}
db.SetConnMaxLifetime(time.Minute * 3)
db.SetMaxOpenConns(10)
db.SetMaxIdleConns(10)

txOpt := sql.TxOptions{
    Isolation: sql.LevelReadCommitted,
    ReadOnly:  false,
}
tx, err := db.BeginTx(context.Background(), &txOpt)
if err != nil {
    log.Fatal(err)
}
defer tx.Commit()

rows, err := tx.Query("select * from users")
if err != nil {
    tx.Commit()
    log.Println("Commit the transaction since there is nothing to do here.")
    log.Fatal(err)
}

// Collect all users into list and update their name to be uppercase
// Rollback if any of the update fails
var users []user
for rows.Next() {
    var u user
    err := rows.Scan(&u.id, &u.name, &u.email, &u.createdAt)
    if err != nil {
	    log.Print(err)
    }
    users = append(users, u)
}
rows.Close()
for _, u := range users {
    _, err = tx.Exec("UPDATE users set name = ? where id = ?", strings.ToUpper(u.name), u.id)
    if err != nil {
	tx.Rollback()
	log.Fatalf("Rollback due to error: %v", err)
    }
}

One thing to keep in mind is that using transaction can leads to driver: bad connection easily if we do not use it properly. In the example code above, I have to collect all users from query result, then close the rows before start any updating operation. A transaction holds only a connection so because rows owns the current transaction’s connection, we need to release rows’s connection first, return it back to transaction, before doing any other operations.

Miscs

Refactor scanning rows

Read database result from query rows can be repetitive, we can move this logic into dedicate functions with rows.Scan as a parameter.

func getUserData(scan func(dest ...interface{}) error) (*user, error) {
	var u user
	err := scan(&u.id, &u.name, &u.email, &u.created_at)
	if err != nil {
		return nil, err
	}
	return &u, nil
}

We can pass rows.Scan function to getUserData to get user struct instance back

for rows.Next() {
    u, err := getUserData(rows.Scan)
    if err != nil {
	    log.Print(err)
    }
    log.Println(u)
}

Migration

Database migration can be done by using go-migrate tool. Migration file can be created by using following command.

migrate create -ext sql -dir FOLDER_MIGRATION_FILES -seq TITLE

Then run the migration

migrate -database YOUR_DATABASE_URL -path PATH_TO_YOUR_MIGRATIONS up

Check the Getting Started guide for more details.
